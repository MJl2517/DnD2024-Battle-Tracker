# Разработка

## Перед изменением

1. Определите feature-модуль, которому принадлежит поведение.
2. Сначала добавьте или обновите тест, фиксирующий ожидаемый результат.
3. Не меняйте публичный `TrackerApi` и схему базы неявно.

## Структура кода

- Один компонент отвечает за один экран, панель или законченное взаимодействие.
- Ориентир для компонента: до 250–300 строк. Превышение допустимо только если разделение разрушает связный сценарий.
- Общий UI выносится только после второго реального использования.
- В `utils` размещаются чистые функции; сетевые запросы и SQL там запрещены.
- Импортер нового источника реализует общий контракт и имеет собственные HTML fixtures.

## Проверки

```powershell
npm.cmd run format
npm.cmd run check
```

`check` проверяет форматирование, lint, типы, unit/integration-тесты, покрытие и Electron E2E. Перед релизом дополнительно выполняется `npm.cmd run dist`.

## Безопасность данных

- E2E и интеграционные тесты используют временную или in-memory базу.
- `secret/`, локальные базы, `release/`, `out/` и локальные ярлыки не коммитятся.
- Не запускайте тестовые сценарии с пользовательским каталогом Electron.
